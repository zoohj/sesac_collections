### Layered Architecture 흐름
1. client가 요청을 보낸다(postman)
2. router가 요청을 받는다  
3. router에서 service를 호출한다.
4. service에서 비즈니스 로직을 처리하고 repository를 호출한다.
5. respository에서 data에 대한 CRUD를 처리한다.
6. respository의 return값이 service -> router에 순차적으로 전달되어 응답한다.

### 코드 포함 (detail 생략)

1. client가 요청을 보낸다(postman)
2. router가 요청을 받는다  
    `routers/post_router.py`
```python
@router.post("", response_model=PostDetailResponse, status_code=status.HTTP_201_CREATED)
def create_post(data:PostCreate):
    return post_service.create_post(data)

```
3. router에서 service를 호출한다.
4. service에서 비즈니스 로직을 처리하고 repository를 호출한다.  
    `services/post_service.py`

```python
def create_post(self, data: PostCreate):
    # data에 대한 검증 등등을 시행

    new_post = Post(title=data.title, content=data.content)

    return post_repository.save(new_post)
```
5. respository에서 data에 대한 CRUD를 처리한다.  
    `repositories/post_repository.py`

```python
def save(self, new_post: Post):
    # 데이터 저장
    self.posts.append(new_post)
    return new_post
```
6. respository의 return값이 service -> router에 순차적으로 전달되어 응답한다.
