post(1) : comment(n) 작성할 예정
fk는 comment가 가지고 있다.
comment CRUD
1. create
	1. 정보 : comment_id(안받음), post_id, content
	2. 요청을 만들껀데, 이걸 어떻게 분배해야할까요?
		1. URL : posts/{post_id}/comments
		2. body : content 입력
2. read
	1. 홀로 조회하지 않고 게시글을 상세 조회했을 때 딸려나온다.
	2. read에 대한 로직은 따로 마련하지 않고
	   GET /posts/{post_id}를 수정.
3. update
	1. 정보 : comment_id, content, 
	2. URL : 
		1. /comments/{comment_id}
		2. /posts/{post_id}/comments/{comment_id}
			1. 이게 좀 더 좋은 URL이다 -> 리소스의 계층을 식별할 수 있어서.
4. delete
	1. URL : /posts/{post_id}/comments/{comment_id}
